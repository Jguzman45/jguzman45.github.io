1- The primary difference between flex-direction: row and flex-direction: column lies in the orientation of the main axis, which dictates how flex items are positioned within their container. When using flex-direction: row, which is the CSS default, items are arranged horizontally from left to right, making it the standard choice for desktop navigation bars. In contrast, flex-direction: column rotates the main axis by 90 degrees ($90^\circ$), causing items to stack vertically from top to bottom. For your assignment, this distinction is essential for responsive design; you will typically use a row layout for your desktop navigation to align the logo and links side-by-side, then use a media query to switch to a column layout on mobile devices so the links stack for better readability on smaller screens.
2- Using relative units is fundamental to responsive design because they allow a layout to adapt fluidly to the varying screen sizes and user settings of different devices. Unlike fixed pixels ($px$), which represent a static number of physical screen dots, relative units like %, vh, or rem calculate dimensions based on a parent container, the viewport size, or the root font size. This flexibility ensures that a website remains functional on both a laptop and a mobile phone by preventing elements from becoming too small to read or too wide for the screen, which would cause broken layouts or horizontal scrolling. Furthermore, using rem is a critical accessibility practice; since it scales based on the user's browser font settings, it ensures that your typography remains readable for users who need to increase their default text size
